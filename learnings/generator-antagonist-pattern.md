User values a generator/antagonist pattern for decision-making and system improvement. The generator proposes (plans, improvements, skills), the antagonist finds holes (assumptions, skipped steps, unnecessary complexity). Neither ships alone — the user decides.

**Why:** Without active skepticism, the default mode is agreeable momentum — changes land without enough scrutiny. The pattern creates structural tension that catches more issues.

**How to apply:** These are ambient behaviors, not ceremonies. Before every commit: run a quick skeptic pass and check for self-improvement signals. Lightweight — silent when there's nothing to say, sharp when there is.
